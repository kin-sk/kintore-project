# 筋トレ記録アプリ（全身法×ジム特化） 要件定義・設計書

本ドキュメントは、PWA（Progressive Web App）とSupabase（データベース）を活用した、個人開発用のジム特化型「筋トレ＆体組成記録アプリ」の要件定義および設計をまとめたものである。

---

## 1. アプリケーション概要
* **コンセプト**: 全身法でジムに通うユーザーが、オフライン環境（電波の悪いジムの地下など）でもストレスなく爆速で記録でき、モチベーションを維持するための自分専用アプリ。
* **運用コスト**: 外部サーバー代 **完全無料（0円）**
* **配信形態**: Webアプリ（PWA対応により、スマホのホーム画面にネイティブアプリ風に配置可能）

---

## 2. 技術スタック
* **フロントエンド**: Vue 3 (Composition API) + Vuetify 3 + TypeScript
* **バックエンド・DB**: Supabase (PostgreSQL / サーバーレス・BaaS)
* **グラフ描画**: Chart.js (or vue-chartjs)
* **ホスティング**: Vercel (GitHub連携による自動デプロイ、永久無料枠)
* **アプリ化の仕組み**: PWA (Vite PWA Plugin を想定)

---

## 3. 機能要件（何ができるか）

### ① 筋トレ記録機能
* その日に行った「種目」「重量(kg)」「回数(レップ数)」「セット数」を記録できる。
* **【入力補助】** 種目を選択した際、**前回の同種目の記録（重量・回数）をプレースホルダーとして自動表示**し、1タップで入力が完了するUXを担保。

### ② インターバルタイマー機能
* セット間にボタンをタップすると、あらかじめ設定した時間（例: 90秒/180秒）のカウントダウンが開始される。
* カウントダウン終了時は、ジム内での使用を想定し、音ではなく**「画面のフラッシュ」および「スマホのバイブレーション（Web Vibration API）」**で通知。

### ③ 種目（メニュー）マスタ管理機能
* 「胸」「背中」「脚」などの**部位**と、「バーベル」「ダンベル」「マシン」などの**器具**を掛け合わせた絞り込み検索が可能。
* デフォルトの主要種目のほか、ユーザー自身が新しいオリジナル種目を自由に追加・編集・削除できる。

### ④ 体組成（体重・体脂肪率）管理機能
* 毎日の「体重(kg)」と「体脂肪率(%)」を記録できる（筋トレをしない日でも入力可能）。
* 過去のデータを期間別（1週間、1ヶ月、3ヶ月）に切り替えられる**折れ線グラフ**として可視化する。

### ⑤ カレンダー「実践マーク」連携機能
* 筋トレの記録データが存在する日付に対し、カレンダー上に自動で「目印（青い丸やダンベルのアイコン）」を表示。
* 記録がない日（オフの日）は何も表示せず、一目で今月の継続状況がわかるようにする。

## 4. 画面設計（全4画面）

### ① 【ホーム 兼 カレンダー画面】
* **役割**: アプリ起動時に最初に表示されるメイン画面。
* **構成要素**:
  * 月間カレンダー（筋トレ実施日に自動で実践マークが付与される）
  * カレンダー下部：「今日の体重・体脂肪を入力」ボタン
  * 画面最下部：「今日の筋トレを開始！」という目立つアクションボタン

### ② 【筋トレ実践（記録）画面】
* **役割**: ジムのインターバル中に片手で操作する画面。
* **構成要素**:
  * 種目の選択ドロップダウン（部位・器具のフィルター付き）
  * 重量(kg)・回数(Reps)の入力フォーム（前回値の自動表示）
  * 「セット追加」ボタンおよび記録済みのセット一覧
  * 画面上部に常駐するカウントダウン式のインターバルタイマー

### ③ 【データ分析（グラフ）画面】
* **役割**: 体重および体脂肪率の推移を確認し、ニヤニヤする画面。
* **構成要素**:
  * 期間切り替えボタン（1W / 1M / 3M / ALL）
  * 体重・体脂肪率が2軸で描画された美しい折れ線グラフ（Chart.js）

### ④ 【種目マスタ管理画面】
* **役割**: アプリに登録されている種目の一覧表示およびカスタム追加。
* **構成要素**:
  * 登録済み種目の一覧（部位別・器具別にソート可能）
  * 新規種目の追加ダイアログ（種目名、対象部位、使用器具の入力）

## 5. データベース設計（Supabase / PostgreSQL）

### 5.1 `exercise_masters` （種目マスタテーブル）
ユーザーが選択できる筋トレ種目の定義。

| カラム名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | uuid | PRIMARY KEY, DEFAULT gen_random_uuid() | 識別子 |
| `user_id` | uuid | REFERENCES auth.users | ユーザーID（自分専用だが将来の拡張性考慮） |
| `name` | varchar | NOT NULL | 種目名（例: ベンチプレス） |
| `part` | varchar | NOT NULL | 鍛える部位（胸, 背中, 脚, 肩, 腕, 腹 etc） |
| `equipment` | varchar | NOT NULL | 使用器具（バーベル, ダンベル, マシン, 自重 etc） |
| `created_at` | timestamptz | DEFAULT now() | 作成日時 |

### 5.2 `workout_logs` （筋トレ記録テーブル）
日々の筋トレの実施ログ。1セットごとに1レコード生成。

| カラム名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | bigint | PRIMARY KEY, GENERATED ALWAYS AS IDENTITY | 識別子 |
| `user_id` | uuid | REFERENCES auth.users | ユーザーID |
| `date` | date | NOT NULL, DEFAULT CURRENT_DATE | 実施日 |
| `exercise_id` | uuid | REFERENCES exercise_masters(id) | 種目ID |
| `weight` | numeric | NOT NULL | 重量 (kg) |
| `reps` | integer | NOT NULL | 回数 (レップ数) |
| `set_number` | integer | NOT NULL | セット数 (1セット目、2セット目...) |
| `created_at` | timestamptz | DEFAULT now() | 記録日時 |

### 5.3 `body_compositions` （体組成記録テーブル）
日々の体重・体脂肪率の推移。1日1レコード。

| カラム名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | bigint | PRIMARY KEY, GENERATED ALWAYS AS IDENTITY | 識別子 |
| `user_id` | uuid | REFERENCES auth.users | ユーザーID |
| `date` | date | NOT NULL, UNIQUE | 記録日（1日1回） |
| `weight` | numeric | NOT NULL | 体重 (kg) |
| `body_fat_percentage` | numeric | | 体脂肪率 (%) |
| `created_at` | timestamptz | DEFAULT now() | 記録日時 |

## 6. 開発マイルストーン
1. **フェーズ1**: Supabaseでデータベース（3つのテーブル）を作成する。
2. **フェーズ2**: Vue 3 + Vuetify 3 のローカル環境を立ち上げ、Supabaseと接続する。
3. **フェーズ3**: 種目マスタ画面、筋トレ実践画面を作り、データの読み書きを可能にする。
4. **フェーズ4**: カレンダー連携、インターバルタイマー、Chart.jsによるグラフ画面を実装。
5. **フェーズ5**: Vercelへデプロイし、PWA設定を行ってスマホのホーム画面に導入する。
